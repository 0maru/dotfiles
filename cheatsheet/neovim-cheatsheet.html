Daily Flow

毎日使う起点と移動

自動保存が有効なので、手動保存は切りの良いタイミングで使う程度で足ります。

素の Vim 動作で押さえるところ

h j k l基本移動
w / b / e単語単位で移動
0 / ^ / $行頭 / 先頭文字 / 行末
/pattern前方検索、n / N で次へ / 前へ
u / Ctrl+rUndo / Redo
.直前の変更を繰り返す

j / k は、カウント未指定のときだけ折り返し行に追従するよう調整済みです。

Treesitter テキストオブジェクト

]f / [f次 / 前の関数先頭
]F / [F次 / 前の関数末尾
]c / [c次 / 前のクラス先頭
]a / [a次 / 前の引数
af / if関数 outer / inner を選択
ac / icクラス outer / inner を選択
aa / ia引数 outer / inner を選択
NSpacetcTreesitter Context をトグル

daf, cif, yaa のように operator と組み合わせる運用が前提です。

整形と編集補助

NVSpacecfConform で明示的にフォーマット
保存時JS / TS / JSON / CSS 系は biome で整形
LSP formatConform 対象ファイルでは fallback させない
入力時blink.cmp が補完 popup と signature help を表示
入力時nvim-autopairs が括弧と引用符を補完
HTML / JSXnvim-ts-autotag が閉じタグを補完

自動整形の責務を conform.nvim に寄せており、Biome がいないときに LSP に勝手に逃がさない構成です。

ビジュアル編集

V< / >選択を維持したままインデント
VJ / K選択行を下 / 上へ移動
Ctrl+v矩形選択
q{reg} ... qマクロ録画
@{reg} / @@マクロ再生 / 直前再生
* - cgn - .同一単語を順に置換する定番パターン

この構成では複数カーソルより、Vim の operator とマクロを軸にした方が素直です。
