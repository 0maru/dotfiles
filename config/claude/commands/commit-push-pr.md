## コンテキスト

- 現在のブランチ: !`git branch --show-current`
- Gitステータス: !`git status`
- ステージ済み・未ステージの変更: !`git diff HEAD`
- 最近のコミット（スタイル参考）: !`git log --oneline -5`
- origin/main以降のこのブランチのコミット: !`git log origin/main..HEAD --oneline 2>/dev/null || echo "(コミットなし)"`

## ルール

- コミットメッセージは日本語で書く
- シークレットを含むファイル（.env、credentials等）はコミットしない
- 意図的でない限り、無関係な変更（例: submoduleのポインタ変更）はコミットしない
- 論理的な変更単位でコミットを分割する（過度な分割は避ける）
- コミットすべき変更がない場合は、プッシュとPR作成に進む

## タスク

上記のコンテキストに基づいて、以下の手順を実行する：

1. **ブランチ**: mainブランチにいる場合、内容を説明する名前で新しいブランチを作成する
2. **コミット**: 変更を論理的な単位に分割し、それぞれ個別にコミットする
   - 変更ファイルを目的・ツール・機能ごとにグループ化する
     - 例: aerospace設定の変更、Claude設定の変更、zsh設定の変更は別々のコミットにする
     - 同じ目的に関連する変更（例: 1つのツールに対する複数ファイルの変更）は1つのコミットにまとめてよい
   - 各コミットには充実した日本語コミットメッセージを書く：
     - `git diff` を十分に分析し、何がなぜ変更されたかを理解する
     - タイトル: 簡潔な要約（50文字以内）、既存のスタイルに従う（例: "Brewfile に ou を追加", "aerospaceモニター設定を修正"）
     - 本文（変更が自明でない場合）:
       - 何がなぜ変更されたか
       - diffからの重要な詳細（例: 追加/削除された設定、変更された値）
       - タイトルの繰り返しではなく、将来の読者に役立つコンテキストを追加する
   - すべての変更が1つの論理単位に属する場合は、1つのコミットで構わない
3. **プッシュ**: `-u` フラグを付けてoriginにプッシュする
4. **PR**: `gh pr create` でプルリクエストを作成する。PRは以下の形式にする：
   - 日本語の短いタイトル（70文字以内）で、ブランチ上のすべての変更を要約する
   - 本文は以下の形式を使う：

```
## 概要
- <ブランチ上のすべてのコミットをまとめた1〜3行の箇条書き>

## テスト計画
- [ ] <テスト項目>
```

5. 完了したらPRのURLを返す
6. 上記のすべてを1つのメッセージ内で実行する。可能な限り並列でツールを呼び出す。
