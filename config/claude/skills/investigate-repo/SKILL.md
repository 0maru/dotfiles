# GitHub リポジトリ調査

GitHub リポジトリを `ghq get` でクローンし、Explorer エージェントで分析する。

## 前提条件

- `ghq` がインストール済みであること

## Phase 1: クローン

1. ユーザーの入力から **GitHub URL** と **調査内容** を抽出する
   - URL が未指定の場合は `AskUserQuestion` で確認する
   - 調査内容が未指定の場合は「プロジェクトの全体概要」をデフォルトとする
2. リポジトリをクローンする:
   ```bash
   ghq get <url>
   ```
3. ローカルパスを取得する:
   ```bash
   ghq list -p | grep <owner/repo>
   ```
4. パスが取得できない場合はエラーを報告して終了する

## Phase 2: プロファイリング

Explorer エージェント（thoroughness: "quick"）を起動し、以下を自動取得する:

- 技術スタック（言語、フレームワーク、ビルドツール）
- ディレクトリ構造の概要
- README の要約
- 主要なエントリーポイント

プロンプト例:
```
以下のリポジトリのプロファイリングを行ってください。
パス: <repo-path>

確認事項:
- 使用言語とフレームワーク
- ディレクトリ構造の概要
- README の要約（あれば）
- 主要なエントリーポイントやメインファイル
```

プロファイリング結果をユーザーに簡潔に共有する。

## Phase 3: 深堀り調査

Explorer エージェント（thoroughness: "very thorough"）を起動し、ユーザーの調査内容に基づいて詳細分析を行う。

プロンプト例:
```
以下のリポジトリについて調査してください。
パス: <repo-path>

プロジェクト概要:
<Phase 2 のプロファイリング結果>

調査内容:
<ユーザーが指定した調査内容>
```

## Phase 4: レポート & クリーンアップ

1. 調査結果を整理してユーザーに報告する
2. `AskUserQuestion` でリポジトリの削除を確認する:
   - 選択肢: 「削除する」「残す」
   - 削除する場合: `ghq rm <repo-path>` で削除
   - 残す場合: ghq 管理下に保持される旨を伝える
