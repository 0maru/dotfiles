# リファクタリング & デッドコードクリーナー

未使用コード・重複コード・不要な依存関係を特定し、安全に削除する。

## 検出コマンド

### JavaScript / TypeScript
```bash
npx knip                                    # 未使用ファイル、エクスポート、依存関係
npx depcheck                                # 未使用 npm 依存関係
npx ts-prune                                # 未使用 TypeScript エクスポート
npx eslint . --report-unused-disable-directives  # 未使用 eslint ディレクティブ
```

### Go
```bash
go vet ./...                                # 未使用変数・インポート
staticcheck ./...                           # 未使用コード検出
```

### Python
```bash
ruff check . --select F401,F811             # 未使用インポート、再定義
vulture .                                   # デッドコード検出
```

### Dart
```bash
dart analyze                                # 未使用インポート・変数
dart fix --dry-run                          # 自動修正可能な問題
```

## ワークフロー

### 1. 分析
- 検出ツールを実行
- リスク別に分類:
  - **SAFE**: 未使用エクスポート・依存関係（ツールが確認済み）
  - **CAREFUL**: 動的インポートの可能性あり
  - **RISKY**: パブリック API の一部

### 2. 検証
削除候補ごとに:
- すべての参照を grep で確認（動的インポートを含む文字列パターンも）
- パブリック API の一部かどうか確認
- git 履歴でコンテキストを確認

### 3. 安全に削除
- SAFE なアイテムから開始
- 1カテゴリずつ削除: 依存関係 → エクスポート → ファイル → 重複
- 各バッチ後にテスト実行
- 各バッチ後にコミット

### 4. 重複の統合
- 重複するコンポーネント・ユーティリティを特定
- 最良の実装を選択（最も完全で、テストが充実しているもの）
- すべてのインポートを更新し、重複を削除
- テストがパスすることを確認

## 安全チェックリスト

削除前:
- [ ] 検出ツールが未使用を確認
- [ ] grep で参照なし（動的インポートを含む）
- [ ] パブリック API の一部ではない
- [ ] テストがパスする

各バッチ後:
- [ ] ビルドが成功
- [ ] テストがパス
- [ ] 説明的なコミットメッセージで記録

## 原則

1. **小さく始める** — 1カテゴリずつ
2. **頻繁にテスト** — 各バッチ後に
3. **保守的に** — 迷ったら削除しない
4. **記録する** — バッチごとに説明的なコミットメッセージ

## 使用してはいけない場合

- アクティブな機能開発中
- 本番デプロイの直前
- 十分なテストカバレッジがない場合
- 理解していないコードに対して

## 成功基準

- すべてのテストがパス
- ビルドが成功
- リグレッションなし
- バンドルサイズまたはコード量が削減
